服务器部署

系统要求

硬件环境

  • 应用服务器： 4核8G内存，本地100G硬盘空间，5M网络
  • 数据库服务器： 2核心4G， 200G存储空间（对于公有云用户，建议使用云服务商提供的RDS服务)

>此配置可以应对每秒1000~1500次API请求， 适应绝大部分普通用户。 特殊客户另议。对SPS有特别要求的用户，可以进行集群部署。系统采用前后端分离的开发方式， 前端页面，tomcat逻辑层，后端数据库，都可以分别部署集群。

操作系统

CentOS 6以上或 Ubuntu 14.04以上 64位发行版。

数据库

Mysql 5.7.x

软件环境

  • nginx 1.8 以上
  • oracle jdk 1.8.x （安装包自带）
  • tomcat 8 （安装包自带）
  • bash
  • zip
  • unzip

客户准备

访问地址（URL）

客户提前准备好接入的URL地址，设置DNS A记录，指向应用服务器的IP地址。如果有SLB服务，则指向SLB的IP地址。

数据库访问账号

如果使用的公有云RDS，则准备下面信息：

  • 数据库URL或IP
  • 端口
  • 用户名
  • 密码
  • 数据库名

目录结构及用途

/app
├── deploy            <-- 部署及备份文件
├── jdk1.7.0_79        <-- JDK目录
├── log                <-- 应用及访问日志
└── sites            <-- 应用部署根目录
    └── default         <-- 部署名称。 默认default
        ├── keyfile        <-- 证书及保密文件的上传
        ├── tomcat-core    <-- 核心业务tomcat
        ├── tomcat-pay        <-- 支付回调tomcat
        └── web        <-- 前端根目录
├── uploads            <-- 客户端上传图片和文件
├── oms                <-- 运用端页面
├── portal            <-- 用户自助端页面
└── h5                <-- 微信服务端
